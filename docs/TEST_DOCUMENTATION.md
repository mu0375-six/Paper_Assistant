# ScholarFlow 测试文档

## 1. 测试目标

本测试文档用于验证 ScholarFlow 是否满足课程项目验收要求。测试重点不是证明系统“完全无缺陷”，而是证明核心学习流程可运行、关键数据可保存、异常情况有反馈、AI 相关逻辑有基本可靠性。

核心测试目标：

- 验证页面可正常访问。
- 验证文献导入、PDF 阅读、AI 精读、闯关评分、笔记保存、课堂汇报等核心流程。
- 验证 API 异常、空输入、重复导入、AI 返回异常分数等边界情况。
- 验证已有自动化测试可以重复运行。
- 为课程验收提供测试方法、测试用例和测试结论依据。

## 2. 测试范围

### 2.1 测试对象

| 层级 | 测试对象 |
| --- | --- |
| 前端页面 | 首页、研究检索页、阅读台、笔记写作页 |
| 后端接口 | `/api/status`、`/api/dashboard`、文献、AI 精读、闯关、笔记、写作接口 |
| 业务逻辑 | 示例论文导入、闯关评分、分数归一化、课堂汇报、笔记来源保存 |
| 数据存储 | 本地 JSON 文件读写 |
| 外部依赖 | DeepSeek API、PDF 解析 |

### 2.2 不在当前测试范围内

| 内容 | 原因 |
| --- | --- |
| 多用户并发 | 当前项目为本地课程演示项目 |
| 生产环境安全审计 | 未部署为公网服务 |
| 大规模论文库检索准确率 | 当前检索仍偏演示性质 |
| 长时间稳定性测试 | 课程项目阶段暂不要求 |

## 4. 测试方法

### 4.1 黑盒测试

从用户角度验证输入与输出，不关注内部代码。

适用模块：

- 页面导航。
- 表单输入。
- PDF 上传。
- AI 问答。
- 闯关提交。
- 笔记保存。

### 4.2 白盒测试

针对后端核心函数和边界逻辑编写单元测试。

当前已有白盒测试：

- 示例论文导入幂等。
- 闯关创建和提交。
- AI 返回 `1` 分时的百分制归一化。
- 保存 PDF 选区来源。
- 保存 AI 回答来源。

### 4.3 集成测试

验证前后端接口、业务服务和数据存储能否连通。

典型流程：

```text
导入示例论文
  -> 获取 dashboard
  -> AI 精读
  -> 提交闯关
  -> 生成课堂汇报
  -> 保存笔记
  -> 刷新 dashboard 验证数据存在
```

### 4.4 端到端测试

使用真实浏览器模拟用户操作。

典型流程：

```text
打开 /reader
  -> 查看新手教程
  -> 导入示例论文
  -> 点击 AI 精读
  -> 切换到闯关
  -> 回答第 1 关
  -> 查看评分反馈
  -> 点击重新作答
  -> 生成课堂汇报
```

### 4.5 异常测试

验证错误输入、外部依赖异常、数据缺失时系统是否给出合理提示。

### 4.6 性能测试

记录关键操作耗时，判断是否满足课程演示需求。

### 4.7 安全测试

重点检查 API Key、上传文件类型、本地数据提交和错误信息暴露。

## 8. 异常测试用例

| 编号 | 异常场景 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| ET-01 | 未配置 API Key | 不设置 `DEEPSEEK_API_KEY` 后调用 AI 精读 | 不崩溃，返回 fallback 或错误提示 |
| ET-02 | 上传空文件 | 不选择文件直接上传 | 不发送请求或提示需要 PDF |
| ET-03 | 上传损坏 PDF | 上传无法解析的 PDF | 返回导入失败提示 |
| ET-04 | 查询不存在论文 | 请求不存在的 paper_id | 返回 404 JSON 错误 |
| ET-05 | 闯关空答案 | answer 为空 | 返回 400 或前端提示 |
| ET-06 | AI 返回非百分制分数 | score 为 1，level 为 pass | 自动归一化为合理百分制 |
| ET-07 | 数据文件为空 | JSON 文件为空数组 | 页面正常显示空状态 |
| ET-08 | 网络中断 | AI 请求失败 | 前端显示错误，不刷新整页 |
